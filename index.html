⚛️ Quantum Random Number Generator (QRNG)
Live Backend Real Quantum Circuit

Runs a real quantum circuit on the Java server: each qubit starts at |0⟩, a Hadamard gate creates superposition (|0⟩+|1⟩)/√2, then Born-rule measurement collapses it to |0⟩ or |1⟩ with P=0.5. This is the same algorithm used by IBM Qiskit's statevector simulator. The output is SHA-3-256 hashed to produce quantum-safe key material.

🔵 What is a Qubit?
|+⟩
Superposition state
💻 Classical Bit

Always exactly 0 or 1. Like a light switch. To try all keys, you must try them one by one.

⚛️ Qubit

Can be 0 AND 1 at the same time — called superposition. 50 qubits represent 2⁵⁰ values simultaneously. This is why quantum search is so fast.

🔗 Entanglement

Two qubits can be "linked" so measuring one instantly tells you the state of the other. Shor's Algorithm uses this to find RSA prime factors in polynomial time.[web:51]

🔓 Shor's Algorithm — Why RSA Will Break

RSA security relies on factoring large numbers being hard. A classical computer factoring RSA-2048 would take longer than the age of the universe. Shor's does it in polynomial time O(log³N) — estimated 8 hours on a mature quantum computer.[web:51]

🔑 Grover's Algorithm — Why AES-128 Is Weakened

Grover's algorithm gives a quadratic speedup for brute-force search. AES-128 has 2¹²⁸ possible keys — Grover's reduces this to 2⁶⁴. AES-256 is still safe (2¹²⁸ after Grover).[web:51]

Why does random text not always get 0?
The scanner is rule‑based. It only reduces the score when it finds patterns that match real vulnerabilities. So truly safe code (or non‑code text) will often stay near 100, while vulnerable code drops much lower.[web:49]
